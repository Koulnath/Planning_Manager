📂 Structure Sacrée du Projet

Nous utilisons le pattern MVC. Chaque fichier a sa place précise.

📦 src/main/java/com/g9/model

Les objets (les "choses" que l'app gère). Ex: Planning.java.

📦 src/main/java/com/g9/service

Le cerveau. Calculs, algorithmes, enregistrement. Ex: ConflictChecker.java.

📦 src/main/java/com/g9/view

Tout ce qui est visuel (Boutons, Tableaux). Ex: MainFrame.java.

📦 src/main/resources

Images, sons et fichiers SQL de Sokeng.

📝 Comment modifier le code proprement ?

Exemple : Ajouter un champ "Salle"

Si tu dois changer quelque chose, suis ce flux :

  1. Modifier le Modèle : Ajoute private String salle; dans Planning.java.
  2. Générer Getters/Setters : Clic droit dans l'IDE -> Source -> Generate Getters and Setters.
  3. Mettre à jour le Service : Si c'est une règle de conflit, modifie ConflictChecker.
  4. Mettre à jour la Vue : Ajoute un JTextField pour que Prisca puisse le voir.

⚠️ Conseil : Ne change jamais le nom d'une variable existante sans prévenir tes camarades via WhatsApp !

🌿 Workflow Git Parfait

Pour éviter de casser le travail des autres :

# 1. Toujours revenir sur main et tirer les dernières modifs
git checkout main
git pull origin main

# 2. Créer sa branche de travail
git checkout -b ma-tache-specifique

# 3. Envoyer son travail
git add .
git commit -m "feat: [votre rôle] description du changement"
git push origin ma-tache-specifique

🗄️ Persistance des Données (DAO)

Pour Sokeng et Juvenal : Le système est prêt pour MySQL. Voici comment ça marche :

1. Le Singleton Connexion

Le fichier com.g9.dao.DatabaseConnection centralise la connexion. Sokeng doit y mettre le bon mot de passe.

2. Le Pattern DAO

On n'écrit jamais de SQL dans les services. On utilise CourseDAO.java. L'implémentation actuelle CourseDAOImpl est en mémoire, et doit être passée en JDBC.
